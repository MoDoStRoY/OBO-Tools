- Зависание программы при закрытии формы претензий к дилерам;
- Множественные ошибки в формировании текста шаблонов корректировки платежей;
- Вылет при вводе дробных чисел в поля сумм корректировки платежей;
- Нормализация всех входящих данных, можно хоть вообще не заполнять;
- Удалён блок ошибок в ТТ для корректировки платежей;

* Шаблон не подстраивается под ОС email ("Способ ОС: E-Mail на номер test@tele2.ru");
* Шаблон не подстраивается под ОС соц.сети ("Способ ОС: Соц.сети на номер smm_saransk@tele2.ru");

    - Из текста запроса где не требуется ответ ТМ убрана строка "Что требуется сделать ТМ:";
    - После чтения файла с чек-листом закрывается поток чтения, чтобы дать возможность редактировать файл;
    - При выполнении запроса без загруженных файлов теперь генерируется ошибка с пояснениями;
    - Вероятно, исправлен баг с не копирующимся форматированием. Требует проверки;
    - Более не происходит повторный запрос корректного кода точки при повторном запросе по тому-же ТТ;
    - При повторном появлении ошибки по некорректному коду точки поле теперь не содержит в себе прошлый код точки;

Проблемы:
  DealerRequest:
    * ФИО ТМ/РТМ не со 100% вероятностью конвертируются в адрес email, т.к. происходит обычный перевод в транзлит, без учёта
      того, что email ТМ/РТМ может содержать первую букву отчества между именем и фамилией;
    * В оглавлении запроса с ожиданием ответа не везде где надо проставлен жирный шрифт;
    * Не поддерживается КОП "Отсутствие дубликатов SIM";
    * Не поддерживаются кейсы с кодом точки T2Start;
    * Баг с подгрузкой прошлого некорректного кода точки в редких случаях (пока не отловлен, не понятно в чём дело);
    * Если код точки содержит какие-либо символы кроме чисел - происходит вылет;
