# Version 0.1.6C

Добавлено:
+ Заготовка счётчика косяков;

Исправлено:
- В интерфейсе корректировок платежей теперь для импорта данных не требуется вставлять данные в текстовое поле, данные подтягиваются
в нужные поля и формируют шаблоны по нажатию кнопки "Загрузка из буфера обмена";
- Математические операции больше не сбоят и double корректно округляется до сотых (наконец-то);
- Вместо номеров ЛС теперь используются номера телефонов;
- При отказе в корректировке больше не появляются строки компенсации;
- При сбросе данных теперь сбрасывается и отказ в корректировке;
- Номера телефонов теперь нормализуются и очищаются от лишних символов;
- Номера телефонов более не заменяют друг друга при импорте через чек-лист;
- Восстановлена функциональность интерфейса претензий к дилерам;
- Нормализация сумм;

Баги:
* Выбранные элементы в листах чек-боксов в интерфейсе корректировки платежей не очищаются (когда я уже это исправлю?);
* Периодически ломается буфер обмена УРС, при этом работает как надо на основной системе. Причина не ясна, но, судя по всему,
это связано с взаимодействием буферов обмена основной системы и УРС. Судя по данным от IT ЧКЦ - копирование данных между 
основной системой и УРС не предусмотрено, поэтому, возможно, это баг или фича CITRIX. Однако происходит это редко и, зачастую,
фиксится банальным CTRL+C CTRL+V на УРС (почему-то именно в браузере мазила);
* Вылет если в буфере обмена был посторонний текст без цифр на момент запуска импорта данных. Нормализация в таком случае не может
обработать входящие данные и бросает исключение (в буфере должна быть по меньшей мере одна цифра и одна буква);
* Из номера ТТ не вычищаются посторонние символы при нормализации;
